# Version Unification, Upgrade UX, and Node Detail Ports

Date: 2026-04-11

## Overview

Three related improvements:
1. Unify version source to `package.json`, remove "v" prefix everywhere
2. Replace the upgrade button in node list with a version-comparison indicator
3. Show port information on the node detail page

## 1. Version Source Unification

### Principle

`package.json` `version` field (set to `"1.0.0"`) is the single source of truth for the project version. No "v" prefix anywhere.

### Changes

**`package.json`**
- Change `"version": "0.1.0"` вЖТ `"1.0.0"`

**`Dockerfile`**
- Remove `ARG AGENT_VERSION=dev` from agent-builder stage
- COPY `package.json` into agent-builder stage, extract version with shell command (`node -p "require('./package.json').version"` or grep/sed)
- Use extracted version for `-ldflags "-X main.Version=${VERSION}"` and `agent-version.txt`
- Xray: keep `ARG XRAY_VERSION=v26.3.27` for GitHub download URL, but strip "v" when writing `xray-version.txt` (write `26.3.27`)

**`.github/workflows/release.yml`**
- Trigger: change from `push: tags: ["v*"]` to `push: branches: [main]`
- Read version from `package.json`: `node -p "require('./package.json').version"`
- Check if git tag `$VERSION` exists; if yes, skip entire job
- If tag does not exist: build and push Docker image, then create git tag `$VERSION`
- Needs `contents: write` permission for tagging
- Docker image tags: `1.0.0`, `1.0`, `1`, `latest`
- Pass `AGENT_VERSION=$VERSION` as build-arg to Dockerfile (override the package.json-derived default, keeping both paths working)

**`agent/api/client_test.go`**
- Change test fixture `Version: "v1"` вЖТ `"1"` and assertion accordingly

**`.claude/skills/e2e-test/SKILL.md`**
- Example `VERSION="v1.0.0"` вЖТ `VERSION="1.0.0"`
- Example `vX.Y.Z` вЖТ `X.Y.Z`
- Example `echo -n "v26.3.27"` вЖТ `echo -n "26.3.27"`

**`public/agent/agent-version.txt`** and **`public/xray/xray-version.txt`**
- These are build artifacts, generated by Dockerfile. For local dev, update current files to not have "v" prefix.

### Not Changed

- Agent Go code вАФ pure string equality comparison, format-agnostic
- Frontend display code вАФ renders whatever string it receives
- Xray GitHub download URL вАФ external system uses "v" prefix tags

## 2. Upgrade Button вЖТ Version Indicator

### Current Behavior
- Node list has an "Upgrade" button in the actions column for each row
- Always visible (disabled when node is offline)

### New Behavior

**API**: `GET /api/nodes` response adds a top-level `latestAgentVersion` field, read from `package.json` at runtime.

**Frontend** (`nodes/page.tsx`):
- Remove the "Upgrade" button from the actions column
- In the `agentVersion` column: when `node.agentVersion !== latestAgentVersion` (and node.agentVersion is not null), show a yellow `AlertCircle` icon (from lucide-react) after the version text
- Clicking the icon opens a `Dialog` with:
  - Current version and latest version displayed
  - "Confirm Upgrade" and "Cancel" buttons
  - Confirm triggers `POST /api/nodes/:id/upgrade`
- Batch upgrade in the selection toolbar remains unchanged

## 3. Node Detail Page вАФ Port Information

### Current Behavior
- Port info only visible in node list via Popover
- Node detail page (`/nodes/[id]`) shows no port data

### New Behavior

**API**: `GET /api/nodes/:id` returns a `ports` object with the same structure as the list API:
```typescript
ports: {
  wg: number;
  xray: number[];
  tunnels: number[];
  socks5: number[];
}
```

Extract the port calculation logic from `GET /api/nodes` (list) into a shared utility function so both endpoints reuse it.

**Frontend** (`nodes/[id]/page.tsx`):
- Add a port information section in the "Node Info" Card, after the version fields
- Layout: grouped by type (WG, Xray, Tunnel, SOCKS5), each group shows ports as `font-mono text-xs bg-muted px-2 py-0.5 rounded` badges вАФ same style as the list Popover
- Only show groups that have ports (e.g., hide SOCKS5 section if no SOCKS5 ports)

**Type update**: Add `ports` to the `NodeDetail` type definition.
