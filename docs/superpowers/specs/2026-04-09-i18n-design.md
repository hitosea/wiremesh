# WireMesh 国际化 (i18n) 设计文档

## 概述

为 WireMesh 管理平台添加多语言支持。首批支持中文（zh-CN）和英文（en），架构可扩展更多语言。

## 需求摘要

| 项目 | 决策 |
|------|------|
| 支持语言 | 中文 + 英文，可扩展 |
| 语言切换方式 | Cookie/Header 检测，URL 不变 |
| API 错误处理 | API 返回错误 key，前端翻译 |
| 默认语言 | 跟随浏览器偏好，fallback 到英文 |
| 切换入口 | 顶栏右侧，与主题切换并列 |
| i18n 库 | next-intl（无路由模式） |

## 项目现状

- 34 个页面/布局文件，26 个组件（含 16 个 shadcn/ui）
- 367+ 处硬编码中文字符串
- 80% 客户端组件（"use client"）
- 无任何 i18n 基础设施
- API 错误通过 `src/lib/api-response.ts` 的 `error(code, message)` 返回中文消息
- 前端通过 `json.error?.message` 消费错误，fallback 也是中文
- Toast 通知使用 Sonner 库

## 架构设计

### 1. 基础架构

#### 新增文件结构

```
wiremesh/
├── messages/
│   ├── zh-CN.json              # 中文翻译
│   └── en.json                 # 英文翻译
├── src/
│   ├── i18n/
│   │   ├── request.ts          # next-intl 请求配置
│   │   ├── config.ts           # 语言列表、默认语言常量
│   │   └── locale.ts           # cookie 读写工具函数
│   ├── components/
│   │   └── locale-toggle.tsx   # 语言切换下拉菜单
```

#### next-intl 无路由模式集成

使用 next-intl 的 "without i18n routing" 模式，不需要 `[locale]` 目录、middleware、URL 重写。

**next.config.ts** — 用 `createNextIntlPlugin` 包装：

```ts
import createNextIntlPlugin from "next-intl/plugin";
const withNextIntl = createNextIntlPlugin();
export default withNextIntl({ output: "standalone", ... });
```

**src/i18n/config.ts** — 常量定义：

```ts
export const locales = ["en", "zh-CN"] as const;
export type Locale = (typeof locales)[number];
export const defaultLocale: Locale = "en";
```

**src/i18n/request.ts** — 从 cookie 读取语言：

```ts
import { cookies, headers } from "next/headers";
import { getRequestConfig } from "next-intl/server";
import { defaultLocale, locales } from "./config";

export default getRequestConfig(async () => {
  // 1. Cookie 优先
  const store = await cookies();
  let locale = store.get("locale")?.value;

  // 2. 匹配 Accept-Language
  if (!locale || !locales.includes(locale)) {
    const acceptLang = (await headers()).get("accept-language") || "";
    locale = parseAcceptLanguage(acceptLang, locales); // 解析 Accept-Language，取第一个匹配 locales 的语言（自行实现，无需额外依赖）
  }

  // 3. Fallback
  if (!locale || !locales.includes(locale)) {
    locale = defaultLocale;
  }

  return {
    locale,
    messages: (await import(`../../../messages/${locale}.json`)).default,
  };
});
```

#### 语言检测优先级

1. Cookie `locale`（用户手动选择）
2. `Accept-Language` header（浏览器偏好）
3. Fallback: `en`

### 3. API 错误码改造

**现状**：API 返回中文消息字符串。

```ts
error("VALIDATION_ERROR", "name 为必填项")
```

**改为**：API 返回翻译 key。

```ts
error("VALIDATION_ERROR", "validation.nameRequired")
```

**前端消费方式**：

```tsx
// 之前
toast.error(json.error?.message ?? "删除失败");

// 之后
const te = useTranslations("errors");
const tc = useTranslations("common");
toast.error(te(json.error?.message) ?? tc("operationFailed"));
```

`error.code`（如 `VALIDATION_ERROR`、`NOT_FOUND`）保持不变，仅 `message` 字段从中文描述改为 key。

### 4. 组件改造模式

#### 客户端组件 — useTranslations hook

```tsx
import { useTranslations } from "next-intl";

export default function NodesPage() {
  const t = useTranslations("nodes");
  const tc = useTranslations("common");
  return <Button>{loading ? tc("saving") : tc("save")}</Button>;
}
```

#### sidebar-constants — 改为存 key

```ts
export const NAV_GROUPS = [
  {
    titleKey: "nav.network",
    items: [
      { href: "/dashboard", labelKey: "nav.dashboard", icon: LayoutDashboard },
      { href: "/nodes", labelKey: "nav.nodes", icon: Server },
      // ...
    ],
  },
];
```

侧栏组件渲染时用 `t(item.labelKey)` 翻译。

#### topbar PAGE_TITLES — 改为 key map

```ts
const TITLE_KEYS: Record<string, string> = {
  "/dashboard": "nav.dashboard",
  "/nodes": "nav.nodes",
  // ...
};
```

渲染时用 `t(getPageTitleKey(pathname))` 翻译。

#### 状态标签 — 翻译函数替代硬编码对象

```tsx
// 之前
const STATUS_LABELS = { online: "在线", offline: "离线", error: "异常" };
<span>{STATUS_LABELS[status]}</span>

// 之后
const t = useTranslations("nodes");
<span>{t(`status.${status}`)}</span>
```

### 5. root layout 改造

```tsx
import { getLocale, getMessages } from "next-intl/server";
import { NextIntlClientProvider } from "next-intl";

export default async function RootLayout({ children }) {
  const locale = await getLocale();
  const messages = await getMessages();

  return (
    <html lang={locale} suppressHydrationWarning ...>
      <body ...>
        <ThemeProvider>
          <NextIntlClientProvider locale={locale} messages={messages}>
            {children}
            <Toaster />
          </NextIntlClientProvider>
        </ThemeProvider>
      </body>
    </html>
  );
}
```

## 不做的事情

- 不引入 URL 路径前缀路由
- 不做 middleware 重写
- 不做服务端组件的翻译（当前 server 组件极少且不含 UI 文本）
- 不做 metadata（title/description）的动态国际化（保持 "WireMesh" 不变）
- Agent（Go 二进制）的日志/输出不做国际化
