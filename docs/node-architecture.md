## 4. 多跳线路详细数据流

以线路 3：A(入口) → B(中转) → C(出口) 为例，展示完整的数据包流转：

### 4.1 WireGuard 设备接入

```
  WG 设备              节点 A (入口)           节点 B (中转)          节点 C (出口)       目标
  10.0.0.100           公网: 1.1.1.1          公网: 2.2.2.2         公网: 3.3.3.3

  ┌────────┐           ┌──────────────┐       ┌──────────────┐      ┌──────────────┐   ┌──────┐
  │        │  WG 隧道   │              │ WG 隧道│              │WG 隧道│              │   │      │
  │  设备   │─────────►│  wm-wg0   │──────►│  wm-tun1  │─────►│  wm-tun1  │   │      │
  │        │           │  10.0.0.1    │       │  10.1.0.2    │      │  10.1.0.6    │NAT│ 互联网│
  │        │  :51820   │      ↓       │:51830 │      ↓       │:51831│      ↓       │──►│      │
  │        │           │  iptables    │       │  iptables    │      │  iptables    │eth0│      │
  │        │           │  wm-wg0   │       │  wm-tun1  │      │  wm-tun1  │   │      │
  │        │           │   → tun1     │       │   → tun2     │      │   → eth0     │   │      │
  │        │           │              │       │      ↓       │      │  MASQUERADE  │   │      │
  │        │           │  wm-tun1  │       │  wm-tun2  │      │              │   │      │
  │        │           │  10.1.0.1    │       │  10.1.0.5    │      │              │   │      │
  │        │           │  :51830      │       │  :51831      │      │              │   │      │
  └────────┘           └──────────────┘       └──────────────┘      └──────────────┘   └──────┘
```

### 4.2 Xray 设备接入（同一条线路，仅入口节点不同）

```
  Xray 设备            节点 A (入口)           节点 B (中转)          节点 C (出口)       目标

  ┌────────┐           ┌──────────────┐       ┌──────────────┐      ┌──────────────┐   ┌──────┐
  │        │ VLESS     │              │       │              │      │              │   │      │
  │  设备   │─────────►│  Xray :443   │       │              │      │              │   │      │
  │        │ WS/gRPC  │      │       │       │              │      │              │   │      │
  │        │           │      ▼       │ WG 隧道│              │WG 隧道│              │   │      │
  │        │           │  wm-wg0   │──────►│  wm-tun1  │─────►│  wm-tun1  │NAT│ 互联网│
  │        │           │  (Xray 转发   │       │  10.1.0.2    │      │  10.1.0.6    │──►│      │
  │        │           │   到 wm-  │:51830 │      ↓       │:51831│      ↓       │eth0│      │
  │        │           │              │       │  iptables    │      │  iptables    │   │      │
  │        │           │              │       │  tun1→tun2   │      │  tun1→eth0   │   │      │
  └────────┘           └──────────────┘       └──────────────┘      └──────────────┘   └──────┘

  Xray 在入口节点将 VLESS 流量解密后注入 wm-wg0，后续链路与 WireGuard 设备完全相同。
```

### 4.3 关键信息

```
  密钥关系:
  ├─ wm-wg0: 设备用节点的 wg_public_key（WG 设备直连）
  │             Xray 设备用 xray_uuid 连接 Xray 服务，Xray 内部转发到 wm-wg0
  ├─ A↔B 隧道: line_nodes 表中 A 和 B 各自的 tunnel_wg_private_key/public_key
  └─ B↔C 隧道: line_nodes 表中 B 和 C 各自的 tunnel_wg_private_key/public_key

  IP 地址空间（两套独立网段，互不冲突）:
  ├─ 10.0.0.0/24  — 设备接入网段（wm-wg0）
  ├─ 10.1.0.0/30  — A↔B 隧道（wm-tun1）
  └─ 10.1.0.4/30  — B↔C 隧道（wm-tun2）

  端口分配:
  ├─ :51820 — wm-wg0（设备接入）
  ├─ :51830 — A↔B 隧道（A 的 wm-tun1 / B 的 wm-tun1）
  └─ :51831 — B↔C 隧道（B 的 wm-tun2 / C 的 wm-tun1）
```

## 6. 节点文件系统结构

```
/
├── etc/
│   ├── wiremesh/
│   │   ├── agent.yaml                     # Agent 配置（server_url, node_id, token）
│   │   └── wireguard/
│   │       ├── wm-wg0.conf             # 设备接入主接口（Peers 由 Agent 动态管理）
│   │       ├── wm-tun1.conf            # 线路隧道接口（Agent 动态创建/销毁）
│   │       ├── wm-tun2.conf            # 线路隧道接口（Agent 动态创建/销毁）
│   │       └── ...
│   │
│   └── systemd/system/
│       └── wiremesh-agent.service            # Agent systemd 服务定义
│
├── usr/local/bin/
│   └── wiremesh-agent                        # Agent Go 二进制
│
└── usr/local/etc/xray/                    # Xray 配置（如启用）
    └── config.json

注意：所有 WireMesh 的 WireGuard 配置放在 /etc/wiremesh/wireguard/（而非系统的 /etc/wireguard/），
     接口名使用 wm- 前缀（而非 wg0/wg1），避免与系统已有 WireGuard 配置冲突。
```
