## 概览

WireMesh 是一个 WireGuard 网状 VPN 管理平台，供内部使用。通过统一的 Web 界面，你可以管理所有 VPN 节点、客户端设备、网络线路和分流规则。

**核心概念：**

- **节点** — 运行 `wiremesh-agent` 的 VPN 服务器。每个节点是网络的基础单元，可以在不同线路中充当入口、中转或出口角色。
- **设备** — 连接到 VPN 的客户端终端，例如笔记本电脑、手机。每个设备获得一份 WireGuard、Xray 或 SOCKS5 配置。
- **线路** — 定义流量的多跳路由路径。每条线路有一个入口节点和一个或多个分支，每个分支定义中转和出口节点的链路。设备通过分配线路来决定流量路径。
- **分流规则** — 基于域名或 IP 的流量路由策略，关联到线路分支。支持白名单和黑名单模式，决定流量走向。

## 仪表盘

仪表盘提供系统整体运行状况的快速概览。

### 指标卡片

展示三张汇总卡片：

- **节点** — 总数，以及在线（绿色）、离线（灰色）、异常（红色，仅在数量大于 0 时显示）的细分。
- **设备** — 总数，以及在线（绿色）、离线（灰色）的细分。
- **线路** — 总数，以及活跃（绿色）、停用（灰色）的细分。

### 状态表格

卡片下方有三个表格，便于快速查看近期状态：

- **节点状态** — 列出近期节点的名称、IP 和当前状态。
- **设备状态** — 列出近期设备的名称、协议和状态。
- **节点流量** — 显示每个节点的上传/下载流量，底部有汇总行（仅在有流量数据时显示）。

## 节点管理

### 添加节点

1. 进入「节点」页面，点击「添加节点」。
2. 填写表单：
   - **名称**（必填）— 便于识别的描述性名称。
   - **IP 地址**（必填）— 服务器的公网 IP。
   - **域名**（可选）— 指向该服务器的域名。
   - **WG 端口**（可选）— WireGuard 监听端口，默认 41820。
   - **标签**（可选）— 逗号分隔的标签，用于分类管理。
   - **备注**（可选）— 自由文本备注。
   - **代理起始端口**（可选）— Xray 和 SOCKS5 共享的端口池起始端口，默认 41443。每条线路自动分配独立端口。
   - **Reality 目标网站**（可选）— Xray Reality 伪装目标，需支持 TLS 1.3，默认 `www.microsoft.com:443`。
3. 点击「保存」。

### 安装 Agent

保存后，进入节点的「安装脚本」页面。会看到一条一键安装命令：

```
curl -fsSL 'https://your-platform/api/nodes/{id}/script?token=...' | bash
```

复制并在服务器上以 root 权限执行。脚本会自动下载 `wiremesh-agent`、创建 `/etc/wiremesh/` 配置目录、注册并启动 `wiremesh-agent.service` 服务。

### 确认连接

刷新节点列表，节点状态变为「在线」即表示 Agent 已成功连接，通常在 30 秒内完成。

**防火墙要求：** UDP 41820 端口需对外开放以供设备接入。共享线路的节点之间需开放隧道端口（41830+ /UDP）。

### 查看节点详情

点击节点行进入详情页，可查看：

- **只读信息** — WireGuard 内网地址、公钥、Agent Token、错误信息（如有）。
- **延迟图表** — 平台到节点的往返延迟趋势。
- **流量图表** — 上传/下载流量随时间变化。

也可在此页面编辑节点的名称、IP、域名、WG 端口、Xray / SOCKS5 设置、标签和备注。

### 版本管理

节点列表中的「Agent」和「Xray」列显示各节点当前运行的版本号。Agent 每次状态上报时会携带版本信息。如果版本号显示为「未知」，说明该节点运行的是不支持版本上报的旧版 Agent，需要手动重跑安装脚本升级到新版。

### 升级 Agent

当管理平台部署了新版本的 Agent 二进制后，可以通过以下方式远程升级节点：

- **单个升级** — 在节点列表中点击对应节点的「升级」按钮。
- **批量升级** — 勾选多个节点后点击「全部升级」。批量升级会分批推送（每批 5 个，间隔 3 秒），避免所有节点同时下载。

升级过程中节点状态显示为「升级中」。Agent 收到升级指令后会自动下载新二进制、校验 SHA256、备份旧版本到 `.backup` 文件、替换后优雅重启。升级完成后节点自动恢复在线。

如果升级失败，可以 SSH 到节点手动回滚：

```
mv /usr/local/bin/wiremesh-agent.backup /usr/local/bin/wiremesh-agent
systemctl restart wiremesh-agent
```

> **注意：** 仅在线节点可以远程升级。离线节点需要手动到服务器上重跑安装脚本。

### 删除节点

在列表页通过删除按钮删除单个节点，或勾选多个节点使用「批量删除」。删除操作会同时触发远程卸载：

- **节点在线时** — 管理平台通过 SSE 通知 Agent 执行卸载脚本，自动清理所有 WireMesh 组件（服务、接口、规则、配置文件、二进制），完成后从数据库中移除节点记录。
- **节点离线时** — 节点被标记为「待删除」。Agent 下次上线时会自动执行卸载。如果节点 7 天内未上线，记录将被自动清理。

### 手动卸载 Agent

如需在不通过管理平台的情况下手动卸载（例如节点无法连接管理平台），SSH 到服务器执行：

```
curl -fsSL 'https://your-platform/api/uninstall-script' | bash
```

这是一个通用脚本，适用于任何节点，无需认证。脚本会停止并移除 `wiremesh-agent` 和 `wiremesh-xray` 服务，拆除所有 WireGuard 接口（`wm-wg0`、`wm-tun*`），清理 iptables 规则、ip 策略路由表、ipset，并删除 `/etc/wiremesh/` 目录和 Agent 二进制文件。系统级软件包（wireguard、iptables、ipset）不会被移除，因为它们可能被其他程序使用。

## 设备管理

### 添加设备

1. 进入「设备」页面，点击「添加设备」。
2. 填写表单：
   - **设备名称**（必填）。
   - **协议**（必填）— 选择 **WireGuard**、**Xray** 或 **SOCKS5**。
   - **线路**（可选）— 将设备分配到某条线路，也可不分配。
   - **标签**（可选）— 逗号分隔的标签。
   - **备注**（可选）。
3. 点击「保存」。系统自动分配 IP 并生成密钥。

### 获取配置

进入设备的「配置」页面：

- **查看** — 完整配置以代码块形式展示。
- **复制** — 复制配置到剪贴板。
- **下载** — 下载配置文件。
- **扫描二维码** — 在移动端直接扫码导入。

Xray 设备额外提供：
- **分享链接** — VLESS URI，可一键导入客户端。
- **Shadowrocket** — 手动配置字段（地址、端口、UUID、TLS 设置等）。
- **Clash Meta** — 适用于 Clash 兼容客户端的 YAML 配置。

SOCKS5 设备显示代理地址（`socks5://用户名:密码@地址:端口`），在系统或浏览器代理设置中配置即可使用。

### 查看设备详情

点击设备查看详情页，包括协议信息、WireGuard 地址/公钥、Xray UUID 或 SOCKS5 代理地址。可编辑设备名称、标签、备注和线路分配。

### 删除设备

在列表页通过删除按钮删除，或勾选多个设备使用「批量删除」。还支持批量切换线路分配。

## 线路编排

线路定义了流量从入口到出口的完整路径，支持多跳中转和多分支。

### 创建线路

1. 进入「线路」页面，点击「创建线路」。
2. 填写**线路名称**、可选标签和备注。
3. 选择**入口节点** — 设备接入的节点。
4. 添加一个或多个**分支**：
   - 每个分支有**名称**和节点链路（中转节点 + 出口节点）。
   - 点击「添加中转」插入中转节点。
   - 为每个分支选择关联的**分流规则**。
   - 标记一个分支为**默认分支** — 不匹配任何规则的流量走默认分支。
5. 点击「保存」。

系统自动从隧道子网（10.211.0.0/16）分配 /30 子网，生成 WireGuard 密钥对，分配隧道端口（从 41830/UDP 起），并将配置推送到所有相关节点。

### 管理线路

- **启用/停用** — 在详情页切换线路状态（活跃/停用）。停用后节点停止转发该线路流量，但不删除配置。
- **线路详情** — 展示入口节点及状态、分支拓扑（节点链路和关联规则）、隧道信息表（跳数索引、源/目标节点、WireGuard 地址和端口）、关联设备数量。
- **删除** — 从所有节点移除隧道配置。

## 分流规则

分流规则通过匹配域名或 IP 来控制流量路由。每条规则以**白名单**或**黑名单**模式运行，并关联到线路分支。

### 规则格式

所有规则字段都是纯文本、每行一条。以 `#` 开头的行视为注释，空行自动忽略：

- **IP 规则** — 每行一条单 IP（如 `8.8.8.8`）或 CIDR 网段（如 `10.0.0.0/8`），精确匹配。
- **域名规则** — 每行一个域名，按**后缀匹配**所有子域名。填 `openai.com` 会同时命中 `chat.openai.com`、`api.openai.com`。不支持前缀、关键词或正则。
- **外部规则源** — URL 指向的纯文本文件，每行一条 CIDR 或域名，Agent 自动识别并分类。**不支持** dnsmasq 格式（`server=/...`）、Clash/Shadowrocket 标签语法（`DOMAIN-SUFFIX,xxx`）、v2ray 的 `geoip.dat` / `geosite.dat` 二进制。
- **模式** — 白名单：匹配的流量走此分支；黑名单：匹配的流量禁止走此分支。

规则编辑页顶部的「规则格式说明」面板也列出了完整约定，可随时展开查看。

### 创建规则

1. 进入「分流规则」页面，点击「添加规则」。
2. 填写表单：
   - **规则名称**（必填）。
   - **模式**（必填）— **白名单**或**黑名单**。
   - **IP 规则**（可选）— 每行一条 IP/CIDR。
   - **域名规则**（可选）— 每行一个域名。
   - **外部规则源**（可选）— 自动拉取规则的远程 URL。
   - **关联分支** — 选择此规则应用到哪些线路分支。**至少绑定一个分支**才能让规则生效，且外部规则源需要通过分支所在线路的入口节点才能触发同步。
   - **备注**（可选）。
3. IP 规则、域名规则、外部规则源至少填写一项。
4. 点击「保存」。

### 管理规则与同步反馈

在详情页可编辑所有字段。配置了外部规则源后，页面会显示同步状态反馈：

- **上次同步时间** — 首次同步成功后记录，从未同步时显示「从未同步」。
- **同步状态徽章** — `✓ 同步成功` 右侧显示本次解析到的 IP 与域名条数；`✗ 同步失败` 下方给出具体错误原因（HTTP 状态码、连接失败等）。
- **立即同步按钮** — 点击后按钮变灰显示「同步中...」并出现旋转 spinner，Agent 从 URL 拉取完成后通过 SSE 实时回写结果，**无需刷新页面**即可看到最新状态。
  - 规则尚未绑定任何分支 → 立即提示「该规则尚未绑定到任何线路分支，无法同步」。
  - 绑定分支所在线路没有入口节点 → 提示「关联分支所在线路没有入口节点，无法触发同步」。
  - 45 秒内未收到 Agent 回调（通常入口节点 Agent 离线）→ 自动超时并提示检查 Agent 状态。

## 系统设置

进入「设置」页面调整全局参数。设置按组分类：

### WireGuard

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| 默认端口 | 41820 | 设备接入的 WireGuard 监听端口（wm-wg0） |
| 默认子网 | 10.210.0.0/24 | 节点和设备的 IP 分配范围 |
| 默认 DNS | 1.1.1.1 | 下发到设备配置的 DNS 服务器 |
| 节点 IP 起始 | 1 | 节点在子网中的 IP 偏移起始值 |
| 设备 IP 起始 | 100 | 设备在子网中的 IP 偏移起始值 |

### 隧道

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| 子网 | 10.211.0.0/16 | 节点间隧道的 IP 范围（每条隧道 /30） |
| 端口起始 | 41830 | 节点间隧道的起始端口 |

### 分流

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| 同步间隔 | 86400 | 自动拉取来源 URL 的间隔（秒） |
| 上游 DNS | 8.8.8.8,1.1.1.1 | 域名规则解析使用的上游 DNS |

### 修改密码

在设置页面可修改登录密码。需输入当前密码、新密码（最少 6 位）并确认。

### 审计日志

审计日志位于「设置 → 审计日志」（独立页面），记录所有管理操作的时间、类型和操作对象。

## 常见问题

### 节点显示离线

1. **检查 Agent 服务**：在节点服务器执行 `systemctl status wiremesh-agent.service`，确认服务正在运行。
2. **查看日志**：执行 `journalctl -u wiremesh-agent.service -n 50` 查看错误信息。
3. **检查防火墙**：确认服务器出站连接未被拦截，Agent 需要能访问管理平台。
4. **重启服务**：`systemctl restart wiremesh-agent.service`。

### 设备无法连接

1. **检查配置**：确认使用的是最新下载的配置，旧配置可能已失效。
2. **检查入口节点**：入口节点必须在线，设备才能接入。
3. **检查端口**：确认入口节点的 41820/UDP（或 Xray 的 41443/TCP）对外可访问。
4. **确认时间同步**：WireGuard 对系统时间敏感，确保客户端和服务端时间误差在合理范围内。

### 线路不通

1. **检查所有节点**：线路中每个节点（入口、中转、出口）都必须在线。
2. **检查隧道端口**：各节点之间的 UDP 隧道端口（41830 起）需要双向可访问，注意云厂商安全组规则。
3. **查看线路详情**：隧道信息表会显示每一跳的状态，定位问题段落。
4. **重新推送配置**：将线路设为停用再重新启用，触发配置重新下发。

### 如何修改网段设置

在「系统设置」页面修改后保存。修改网段或端口会影响现有配置，节点会在下次同步时收到更新，设备配置需重新下载。
