## 概览

WireMesh 是一个 WireGuard 网状 VPN 管理平台，供内部使用。通过统一的 Web 界面，你可以管理所有 VPN 节点、客户端设备、网络线路和分流规则。

**核心概念：**

- **节点** — 运行 `wiremesh-agent` 的 VPN 服务器。每个节点是网络的基础单元，可以在不同线路中充当入口、中转或出口角色。
- **设备** — 连接到 VPN 的客户端终端，例如笔记本电脑、手机。设备通过入口节点接入网络。
- **线路** — 定义流量的多跳路由路径：入口节点 → 可选中转节点 → 出口节点。同一节点可同时参与多条线路。
- **分流规则** — 基于域名或 IP 的流量路由策略。根据目标地址决定走哪条线路，或直连不走 VPN。

## 仪表盘

仪表盘提供系统整体运行状况的快速概览。

### 指标卡片

| 指标 | 说明 |
|------|------|
| 节点总数 | 已注册的节点数量 |
| 在线节点 | Agent 正常上报、状态健康的节点数 |
| 设备总数 | 已添加的客户端设备数量 |
| 在线设备 | 当前活跃连接的设备数 |
| 线路总数 | 已创建的线路数量 |
| 活跃线路 | 当前启用且所有节点在线的线路数 |

### 状态指示

- **在线（绿色）** — Agent 正常运行，最近心跳正常
- **离线（灰色）** — Agent 未响应或服务已停止
- **异常（红色）** — Agent 在线但检测到错误，例如 WireGuard 接口异常

### 近期活动

展示最近的系统操作记录，包括节点上线/下线、设备添加、线路变更等事件，便于快速了解系统动态。

## 节点管理

### 添加节点

1. 进入「节点」页面，点击「添加节点」
2. 填写节点名称（便于识别的描述性名称）、公网 IP 地址，以及可选备注
3. 提交后系统会生成一条安装命令

### 安装 Agent

复制安装脚本，在目标服务器上以 root 权限执行：

```bash
curl -fsSL https://your-platform/install.sh | bash -s -- --token <节点Token>
```

脚本会自动完成以下操作：
- 下载并安装 `wiremesh-agent` 二进制
- 注册并启动 `wiremesh-agent.service` systemd 服务
- 创建配置目录 `/etc/wiremesh/`

### 确认连接

安装完成后，刷新节点列表。节点状态变为「在线」即表示 Agent 已成功连接平台。通常在 30 秒内完成。

### 查看节点详情

点击节点行可查看详情面板，包括：
- **状态历史** — 近期在线/离线变化记录
- **延迟图表** — 平台到节点的往返延迟趋势
- **流量图表** — 入流量和出流量统计

### 删除节点

删除节点前，系统会提示该节点关联的线路。建议先从相关线路中移除该节点，再执行删除操作，避免线路中断。

## 设备管理

### 添加设备

1. 进入「设备」页面，点击「添加设备」
2. 填写设备名称
3. 选择**入口节点** — 设备流量将从该节点进入网络
4. 选择连接协议：
   - **WireGuard** — 适合支持原生 WireGuard 的系统（Linux、macOS、Windows、iOS、Android）
   - **Xray VLESS Reality** — 适合需要流量伪装的场景，流量特征更难被检测

### 获取配置

添加成功后，可以：
- **下载配置文件** — 导入到对应的客户端应用
- **扫描二维码** — 在移动端直接扫码导入

### 查看设备详情

点击设备行查看详情：
- **流量统计** — 累计上传/下载流量
- **在线状态** — 最近一次活跃时间

### 删除设备

删除设备会立即撤销其 VPN 访问权限。入口节点会在下次配置同步时移除该设备的 WireGuard Peer。

## 线路编排

线路定义了流量从入口到出口的完整路径，支持多跳中转。

### 创建线路

1. 进入「线路」页面，点击「创建线路」
2. 填写线路名称
3. 按顺序选择节点：
   - **入口节点** — 客户端设备接入的节点（必选）
   - **中转节点** — 可选，可添加一个或多个中间跳
   - **出口节点** — 流量最终出网的节点（必选）

### 自动配置

创建线路后，系统自动完成：
- 为每段隧道分配 `10.211.0.0/16` 网段内的 `/30` 子网
- 生成 WireGuard 隧道密钥对
- 分配隧道端口（从 41830 起依次递增）
- 将配置推送到各节点 Agent

### 启用与停用

线路支持随时启用或停用，不影响线路配置。停用后相关节点会停止转发该线路的流量。

### 查看线路状态

线路详情展示各跳的连通性状态，帮助定位断点：
- 绿色表示该段隧道正常
- 红色/灰色表示该段存在问题

### 删除线路

删除线路会通知所有相关节点清理对应的 WireGuard 接口和 iptables 规则。如有分流规则关联到该线路，需先解除关联。

## 分流规则

分流规则决定特定流量走哪条线路，或直连不经过 VPN。

### 创建规则

1. 进入「分流规则」页面，点击「添加规则」
2. 填写规则名称
3. 选择**匹配模式**：
   - **域名后缀** — 匹配该域名及其所有子域名，例如 `example.com` 匹配 `www.example.com`
   - **域名关键词** — 匹配包含指定关键词的域名
   - **IP CIDR** — 匹配指定 IP 段，例如 `192.168.0.0/16`
4. 填写匹配内容
5. 设置**优先级**（数字越小优先级越高，建议以 10 为步长留出调整空间）
6. 关联到目标**线路**，或选择「直连」

### 规则评估顺序

系统按优先级从小到大依次匹配，命中第一条规则后立即执行对应动作，不再继续匹配后续规则。建议将更精确的规则设置更高优先级（更小的数字）。

### 启用与停用

规则支持单独启用/停用，便于临时调整路由策略而无需删除规则。

## 系统设置

### 网段配置

- **设备子网** — 客户端设备分配 IP 的网段，默认 `10.210.0.0/24`。节点从 `.1` 起分配，设备从 `.100` 起分配
- **隧道子网** — 节点间隧道使用的网段，默认 `10.211.0.0/16`，每条隧道使用 `/30` 子网

> 修改网段后，需要重新下发配置到所有受影响的节点。

### 端口配置

| 用途 | 默认值 | 协议 |
|------|--------|------|
| 设备接入（wm-wg0） | 41820 | UDP |
| 隧道端口起始（wm-tun*） | 41830 | UDP |
| Xray 端口起始 | 41443 | TCP |

每条新线路/接口自动在起始端口基础上递增分配。

### DNS 设置

配置 VPN 客户端使用的 DNS 服务器，用于设备接入时下发 DNS 配置。

### 审计日志

记录所有管理操作，包括操作时间、操作类型和操作对象，用于追踪配置变更历史。

## 常见问题

### 节点显示离线

1. **检查 Agent 服务状态**：在节点服务器执行 `systemctl status wiremesh-agent.service`，确认服务正在运行
2. **检查防火墙**：确认服务器出站连接未被防火墙拦截，Agent 需要能访问管理平台
3. **查看日志**：执行 `journalctl -u wiremesh-agent.service -n 50` 查看最近错误
4. **重启服务**：`systemctl restart wiremesh-agent.service`

### 设备无法连接

1. **检查配置文件**：确认使用的是最新下载的配置，旧配置可能已失效
2. **检查入口节点状态**：入口节点必须在线，设备才能接入
3. **检查端口**：确认入口节点的 41820/UDP 端口（或 Xray 的 41443/TCP）对外可访问
4. **确认时间同步**：WireGuard 对系统时间敏感，确保客户端和服务端时间误差在合理范围内

### 线路不通

1. **检查所有节点在线**：线路中每个节点（入口、中转、出口）都必须在线
2. **检查隧道端口**：各节点之间的 UDP 隧道端口（41830 起）需要双向可访问，特别注意云厂商安全组规则
3. **查看线路状态**：线路详情页会标出哪一跳出现问题，重点排查红色标注的段
4. **重新推送配置**：在线路详情页触发「重新下发配置」，让 Agent 重新应用 WireGuard 配置

### 如何修改网段设置

在「系统设置」页面修改网段配置后，需要在每个受影响的节点上重新下发配置。操作步骤：
1. 前往系统设置修改网段
2. 进入节点列表，对每个节点触发「重新下发配置」
3. 重新下载所有设备的 VPN 配置文件，因为设备 IP 分配会发生变化
